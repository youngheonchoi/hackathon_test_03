# 어떡해? MVP 기획서

## 1. 프로젝트 개요

### 프로젝트명

**어떡해?**

### 한 줄 설명

사용자의 고민을 해결해주는 척하지만, 실제로는 “어떡해...”를 반복하며 걱정을 무한히 증폭시키는 B급 감성 AI 역상담 서비스.

### 핵심 컨셉

일반적인 AI 상담 서비스는 사용자의 고민을 정리하고 해결책을 제시한다.
하지만 **어떡해?**는 사용자의 고민을 듣고, 공감하는 척하다가 계속 새로운 걱정 가능성을 던진다.

사용자는 AI가 제공하는 선택지를 고르며 대화를 이어가지만, 대화가 길어질수록 화면과 UI가 점점 망가지고, 결국 사용자는 한숨이 나오는 경험을 하게 된다.

마지막에는 처음 입력했던 고민 문장이 단어 조각으로 분리되고, 사용자는 그 조각들을 쓰레기통에 버리며 서비스를 마무리한다.

## 2. MVP 목표

이번 MVP의 목표는 모든 기능을 완벽하게 구현하는 것이 아니라, 아래의 핵심 경험을 확실하게 전달하는 것이다.

> “AI가 고민을 해결해줄 줄 알았는데, 오히려 걱정이 늘어나고 화면까지 망가진다.”

MVP에서 반드시 보여줘야 하는 핵심은 다음과 같다.

1. 사용자가 고민을 입력한다.
2. AI가 해결책 대신 걱정을 증폭시키는 꼬리질문을 한다.
3. 사용자는 AI가 제공한 선택지 중 하나를 선택한다.
4. 대화가 이어질수록 UI/UX가 점점 망가진다.
5. 사용자가 일정 시간 선택하지 않으면 탈출 버튼이 나타난다.
6. 마지막 페이지에서 고민 단어 조각을 쓰레기통에 버린다.
7. 결과보기 버튼을 통해 걱정인형 입양 결과 이미지를 다운로드할 수 있다.

## 3. 공통 UI / UX

모든 페이지에는 아래 요소가 공통으로 존재한다.

### 떠다니는 고민 텍스트

화면 전체에 여러 사람들의 고민 문장이 둥둥 떠다닌다.

예시:

* 내일 발표 어떡하지
* 돈이 없다 어떡해
* 소개팅 망하면 어떡하지
* 과제 아직 못 했는데 어떡해
* 답장 안 오면 어떡하지
* 잠이 안 와 어떡해

요구사항:

* 고민 문장들은 화면 곳곳에서 천천히 움직인다.
* 크기, 속도, 투명도, 회전값이 조금씩 다르면 좋다.
* 페이지가 바뀌어도 배경 분위기는 유지된다.
* 대화가 길어질수록 고민 텍스트 수가 증가하거나 움직임이 과격해질 수 있다.

### BGM

직접 녹음한 “어떡해... 어떡해...” 음성이 배경음으로 무한 반복된다.

요구사항:

* 모든 페이지에서 재생된다.
* 사용자가 처음 진입했을 때 브라우저 정책상 자동재생이 막힐 수 있으므로, 첫 클릭 이후 재생되도록 처리한다.
* 반복 재생된다.

### BGM 끄기 버튼

오른쪽 상단에 BGM 끄기 버튼이 있다.

하지만 사용자가 마우스를 버튼 근처로 가져가면 버튼이 자석처럼 도망간다.

요구사항:

* 버튼 텍스트 예시: `BGM 끄기`
* 마우스가 가까워지면 버튼 위치가 랜덤하게 이동한다.
* 실제로 끌 수 없거나, 끄기 매우 어렵게 만든다.
* MVP에서는 “마우스 오버 시 버튼 이동”까지만 구현해도 충분하다.

### 커스텀 커서

기본 마우스 커서 대신 서비스 분위기에 맞는 커스텀 커서를 사용한다.

예시:

* 작은 걱정인형 얼굴
* `?` 모양 커서
* 한숨 표시 커서

### 페이지 전환 효과

모든 페이지는 딱딱하게 즉시 전환되지 않고, 자연스럽게 페이드 인 / 페이드 아웃되며 이동한다.

요구사항:

* 페이지 이동 시 현재 화면이 서서히 사라진다.
* 다음 화면이 서서히 나타난다.
* 전환 시간은 약 300ms ~ 700ms 정도가 적당하다.
* 전체적으로 부드럽지만, B급 감성에 맞게 살짝 과장된 흔들림이나 흐림 효과를 추가해도 좋다.
* React 사용 시 `framer-motion`, CSS transition, route transition 등을 사용할 수 있다.

예시 효과:

* opacity 0 → 1
* blur 8px → 0
* scale 0.98 → 1
* 화면이 살짝 흔들리며 등장

## 4. 페이지 구성

### 메인 페이지

목적: 서비스의 컨셉을 처음 보여주고, 사용자가 고민 입력 단계로 들어가게 한다.

화면 요소:

* 어떡해 걱정인형 캐릭터 PNG
* 프로젝트 제목: `어떡해?`
* 소제목 / 설명 문구
* `나 고민이 있어` 버튼
* 공통 배경 요소

캐릭터 동작:

* 사용자가 마우스를 움직이면 걱정인형 캐릭터 이미지가 여러 포즈로 교차되며 바뀐다.
* 캐릭터 PNG를 여러 장 준비한다.
* 마우스 이동 이벤트가 발생할 때마다 이미지가 순환되거나 랜덤으로 변경된다.

문구 예시:

```text
어떡해?
```

```text
고민을 해결해주는 척하다가 더 걱정하게 만드는 AI 걱정 서비스
```

```text
나 고민이 있어
```

### 고민 입력 페이지

목적: 사용자가 자신의 고민을 입력하고 AI 걱정 대화로 진입한다.

화면 요소:

* 걱정인형 캐릭터
* 고민 입력 input
* 제출 버튼
* 공통 배경 요소

입력창 placeholder 예시:

```text
고민을 입력하세요... 어차피 해결은 안 됩니다
```

버튼 문구 예시:

```text
고민 맡기기
```

동작:

1. 사용자가 고민을 입력한다.
2. 버튼을 누르면 고민 내용을 저장한다.
3. AI 걱정 대화 페이지로 이동한다.
4. 이동 시 페이드 아웃 / 페이드 인 전환 효과를 적용한다.

### AI 걱정 증폭 대화 페이지

목적: 사용자의 고민을 해결하는 척하면서, 실제로는 걱정을 계속 증폭시킨다.

기본 흐름:

1. 사용자가 입력한 고민을 기반으로 AI가 고민 유형을 분석한다.
2. AI가 “어떡해...” 식의 공감 문장을 출력한다.
3. AI가 새로운 걱정 꼬리질문을 던진다.
4. 사용자는 AI가 제공한 선택지 2~3개 중 하나를 선택한다.
5. 선택 후 AI는 다시 더 걱정스러운 꼬리질문을 던진다.
6. 이 과정은 기본적으로 무한 반복된다.
7. 질문과 답변이 한 번씩 끝날 때마다 UI/UX가 점점 망가진다.
8. 사용자가 5초간 선택지를 누르지 않으면 탈출 버튼이 나타난다.
9. 탈출 버튼을 누르면 마지막 페이지로 이동한다.

AI 말투:

* “어떡해...”
* “그랬구나...”
* “근데 만약에...”
* “혹시 이런 경우는 생각해봤어...?”
* “그것도 안 되면 어떡해...?”

대화 예시:

```text
사용자 고민:
소개팅 나가는데 너무 긴장돼.

AI:
어떡해... 그랬구나...
근데 만약에 말이 잘 안 나오면 어떡해...?

선택지:
1. 다른 얘기를 하면 되지 않을까?
2. 그냥 웃으면 되지 않을까?
3. 잠깐 화장실 다녀온다고 할까?

사용자 선택:
다른 얘기를 하면 되지 않을까?

AI:
어떡해...
근데 만약에 그 얘기도 상대가 방금 전에 했던 얘기면 어떡해...?
```

### 5초 미선택 탈출 버튼

사용자가 선택지를 5초 동안 누르지 않으면 화면 아래에 탈출 버튼이 나타난다.

버튼 문구:

```text
아 그런가...?
```

동작:

* 버튼을 누르면 마지막 페이지로 이동한다.
* 페이지 이동 시 페이드 아웃 / 페이드 인 전환 효과가 적용된다.

## 5. UI/UX 망가짐 단계

대화가 한 턴씩 진행될 때마다 화면이 점점 망가진다.

| 턴 수    | 변화                             |
| ------ | ------------------------------ |
| 1턴     | 화면이 살짝 흔들림                     |
| 2턴     | 배경색이 미묘하게 바뀜                   |
| 3턴     | 텍스트가 조금 흐려짐                    |
| 4턴     | 버튼 위치가 조금씩 어긋남                 |
| 5턴     | 걱정인형 표정이 화난 표정으로 변경            |
| 6턴     | 떠다니는 고민 텍스트 수 증가               |
| 7턴     | 화면 blur 증가                     |
| 8턴     | 폰트 크기와 자간이 이상해짐                |
| 9턴     | UI 요소들이 살짝 회전됨                 |
| 10턴 이상 | 화면 전체가 과하게 흔들리고 “어떡해” 텍스트가 도배됨 |

MVP에서는 아래 정도만 구현해도 충분하다.

* 턴 수에 따라 배경색 변경
* 턴 수에 따라 화면 흔들림 증가
* 턴 수에 따라 blur 증가
* 턴 수에 따라 캐릭터 표정 변경
* 턴 수에 따라 떠다니는 고민 텍스트 증가

## 6. 마지막 페이지

목적: 사용자가 입력했던 고민을 단어 조각으로 분해하고, 직접 쓰레기통에 버리면서 서비스를 마무리한다.

상단 문구:

```text
아직도 그게 고민돼?
```

현실적인 도움 문구 예시:

```text
괜찮아.
해결은 못 했지만,
적어도 이 서비스는 끝났어.
```

고민 단어 조각:

* 처음 입력했던 고민을 띄어쓰기 기준으로 분리한다.

예시:

```text
입력한 고민:
소개팅 나가는데 너무 긴장돼

분리 결과:
소개팅 / 나가는데 / 너무 / 긴장돼
```

드래그 앤 드롭:

* 각 단어 조각은 draggable 요소다.
* 쓰레기통 영역에 drop하면 해당 단어 조각이 사라진다.
* 모든 단어 조각이 사라지면 버튼 2개가 나타난다.

모든 조각을 버린 후 나타나는 버튼:

* `다시하기`
* `결과보기`

## 7. 결과 이미지

컨셉: 사용자가 걱정을 버리고, “어떡해 걱정인형”을 입양했다는 결과 카드 이미지.

이미지에 들어갈 요소:

* 걱정인형 캐릭터
* 프로젝트명 `어떡해?`
* 입양 완료 문구
* 사용자가 입력했던 고민 요약
* 랜덤 걱정인형 이름
* 날짜

문구 예시:

```text
어떡해 걱정인형 입양 완료
```

```text
이 인형은 당신 대신
“어떡해...”를 중얼거립니다.
```

다운로드:

* `결과 이미지 다운로드` 버튼을 누르면 이미지 파일을 저장할 수 있다.
* MVP에서는 HTML 영역을 canvas로 변환한 뒤 다운로드하는 방식도 가능하다.
* 예: `html2canvas` 사용

## 8. AI 기능 요구사항

AI 입력값:

* 사용자가 처음 입력한 고민
* 이전 대화 기록
* 현재 턴 수

AI 출력값 예시:

```json
{
  "concernType": "소개팅/대인관계/불안",
  "aiMessage": "어떡해... 근데 만약에 말이 잘 안 나오면 어떡해...?",
  "choices": [
    "다른 얘기를 하면 되지 않을까?",
    "그냥 웃으면 되지 않을까?",
    "잠깐 화장실 다녀온다고 할까?"
  ]
}
```

AI 프롬프트 방향:

```text
너는 사용자의 고민을 해결하지 않는 AI 걱정인형이다.
사용자의 고민에 공감하는 척하지만, 해결책을 주지 않고 새로운 걱정 가능성을 제시한다.
항상 “어떡해...”라는 말투를 섞는다.
매 응답마다 사용자가 고를 수 있는 선택지 2~3개를 제공한다.
선택지는 해결책처럼 보이지만, 다음 질문에서 다시 걱정으로 이어질 수 있어야 한다.
응답은 JSON으로 반환한다.
```

## 12. MVP 우선순위

### 반드시 구현

* 메인 페이지
* 고민 입력 페이지
* AI 걱정 대화 페이지
* 마지막 정리 페이지
* 페이지 전환 페이드 효과
* 떠다니는 고민 텍스트
* BGM 반복 재생
* 도망가는 BGM 끄기 버튼
* 걱정인형 캐릭터
* 선택지 기반 대화
* 턴 수에 따른 UI 망가짐
* 3초 미선택 시 `아 그런가...?` 버튼 표시
* 고민 단어 조각 드래그 앤 드롭
* 결과 이미지 다운로드

### 시간이 부족하면 축소 가능

* AI API 대신 템플릿 기반 응답 사용
* 결과 이미지는 정적인 카드 디자인으로 대체
* 커스텀 커서는 간단한 CSS cursor로 구현
* UI 망가짐 단계는 3~4단계만 구현
* 드래그 앤 드롭은 클릭해서 버리기로 대체 가능

## 13. 데모 시나리오

1. 사용자가 메인 페이지에 진입한다.
2. 화면에는 여러 고민들이 둥둥 떠다니고, “어떡해...” BGM이 반복된다.
3. 사용자가 BGM을 끄려고 하지만 버튼이 도망간다.
4. 사용자가 `나 고민이 있어` 버튼을 누른다.
5. 페이지가 부드럽게 페이드 전환되며 고민 입력 페이지로 이동한다.
6. 사용자가 고민을 입력한다.
7. AI 걱정 대화 페이지로 이동한다.
8. AI가 고민을 해결하는 척하면서 더 불안한 질문을 던진다.
9. 사용자가 선택지를 고를수록 화면이 점점 망가진다.
10. 사용자가 선택지를 누르지 않고 3초 기다리면 `아 그런가...?` 버튼이 나타난다.
11. 버튼을 누르면 마지막 페이지로 부드럽게 전환된다.
12. 사용자는 자신의 고민 단어 조각을 쓰레기통에 버린다.
13. 모든 조각을 버리면 `다시하기`, `결과보기` 버튼이 나타난다.
14. `결과보기`를 누르면 걱정인형 입양 결과 이미지가 표시된다.
15. 사용자는 결과 이미지를 다운로드한다.

## 14. 최종 요약

**어떡해?**는 고민 해결 AI처럼 보이지만, 실제로는 걱정을 무한히 증폭시키는 B급 감성 역상담 서비스다.

사용자는 고민을 입력하고, AI는 계속 “그럼 이건 어떡해?”라는 식으로 꼬리질문을 던진다.
대화가 길어질수록 UI는 흐려지고, 흔들리고, 망가지며 사용자의 답답함을 시각적으로 표현한다.

사용자가 더 이상 선택지를 고르지 않으면 `아 그런가...?` 버튼으로 탈출할 수 있고, 마지막에는 자신의 고민을 단어 조각으로 분리해 쓰레기통에 버린다.

결국 이 서비스는 고민을 해결하지 않는다.
대신 사용자가 한숨을 쉬게 만들고, 마지막에 아주 작은 해소감을 준다.

## 15. 확정 동작 보완 사항

### 배경 고민 문구

* 모든 화면의 배경에는 최소 수십 개의 서로 다른 고민 문구가 동시에 떠다닌다.
* 문구는 장식으로만 희미하게 처리하지 않는다. 사용자가 읽을 수 있는 명도와 크기를 유지한다.
* 문구의 위치, 크기, 이동 속도를 서로 다르게 하여 화면 전체가 걱정으로 가득 찬 인상을 만든다.

### 무한 질문·답변 세트와 화면 붕괴

* AI 질문 1개와 사용자 답변 1개를 하나의 세트로 정의한다.
* 질문과 답변 과정에는 마지막 세트가 없다. 준비된 고정 질문 묶음을 순환하며 무한히 이어진다.
* API 통신은 사용하지 않고 고정 질문과 고정 선택지를 사용한다.
* 세트가 누적될수록 채도, 대비, 흔들림, 회전, 기울기, 블러, 요소 위치와 글자 크기가 단계적으로 망가진다.
* 마지막 붕괴 단계에서는 답변 선택지가 보이더라도 조작할 수 없어 더 진행할 수 없는 상태가 된다.
* 각 세트가 시작될 때마다 독립적인 3초 타이머를 새로 시작한다.
* 3초 안에 답변하지 않으면 탈출 버튼이 나타난다. 답변하여 다음 세트로 이동하면 이전 타이머를 폐기하고 다시 3초를 센다.
* 탈출 버튼은 화면 붕괴의 영향을 받지 않는 별도 고정 UI로 표시한다. 블러, 회전, 흔들림, 채도 저하가 적용되지 않아 항상 읽고 누를 수 있어야 한다.

### BGM 재생 및 끄기 버튼

* `/voice`의 WAV 3개를 순서대로 재생한다.
* 각 음원이 끝난 뒤 정확히 2초 후 다음 음원을 재생하고, 세 번째 음원 뒤에는 첫 번째 음원으로 돌아간다.
* 재생은 한 번 시작된 뒤 중간에 멈추지 않고 계속 이어져야 하며, 다음 음원으로 넘어갈 때 공백은 2초 외에는 없어야 한다.
* BGM 끄기 버튼은 포인터가 닿기 전에 화면의 다른 임의 위치로 이동한다.
* 마우스, 터치, 키보드 어느 방식으로도 BGM 끄기 버튼을 실제로 누를 수 없어야 한다.
